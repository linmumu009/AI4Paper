# 项目启动指南

这是一个前后端分离的 arXiv 论文自动化筛选与速读系统。

## 📋 前置要求

- Python 3.8+ 
- Node.js 16+ 和 npm
- 已配置的 API Keys（MinerU Token、DashScope Key 等，已在 `Sever/config/config.py` 中配置）

## 🚀 启动步骤

### 第一步：安装后端依赖

在项目根目录下，进入 `Sever` 目录并安装 Python 依赖：

```bash
cd Sever
pip install -r requirements.txt
```

> 💡 建议使用虚拟环境：
> - Windows: `python -m venv .venv && .\.venv\Scripts\activate`
> - Linux/Mac: `python -m venv .venv && source .venv/bin/activate`

### 第二步：安装前端依赖

在项目根目录下，进入 `View` 目录并安装 npm 依赖：

```bash
cd View
npm install
```

### 第三步：启动后端服务器

在 `Sever` 目录下运行：

```bash
cd Sever
uvicorn api:app --reload --port 8000
```

后端 API 服务器将在 `http://localhost:8000` 启动。

### 第四步：启动前端开发服务器

在 `View` 目录下运行（新开一个终端窗口）：

```bash
cd View
npm run dev
```

前端开发服务器将在 `http://localhost:5173` 启动（Vite 默认端口）。

## ✅ 验证启动

1. 打开浏览器访问 `http://localhost:5173`
2. 如果看到前端界面，说明启动成功
3. 后端 API 文档可访问 `http://localhost:8000/docs` 查看

## 📝 其他说明

### 运行论文处理流水线

如果需要运行论文处理流水线（非 Web 界面），可以在 `Sever` 目录下运行：

```bash
cd Sever
python app.py
```

或带参数运行：

```bash
python app.py default --query "LLM alignment" --anchor-tz Asia/Shanghai
```

### 配置说明

主要配置文件位于 `Sever/config/config.py`，包含：
- API Keys（MinerU Token、DashScope Key 等）
- 模型配置
- 数据目录配置

## 🛠️ 常见问题

1. **端口被占用**：修改 `uvicorn` 的 `--port` 参数或 `vite.config.ts` 中的端口配置
2. **依赖安装失败**：检查 Python/Node.js 版本是否符合要求
3. **API 连接失败**：检查 `Sever/config/config.py` 中的 API Keys 是否正确配置
